# FISC Security Guidelines on Computer Systems for Financial Institutions

The dominant security framework for Japanese banks and financial institutions, developed jointly by FISC with the FSA and the Bank of Japan. ~300 controls, 13th edition published November 2025.

- **Owner:** Center for Financial Industry Information Systems (FISC)
- **SecID:** `secid:control/fisc.or.jp/security-guidelines`
- **Source:** https://www.fisc.or.jp/english/
- **License:** members-only / paid — **not publicly redistributable**. See `security-guidelines-metadata.json`.

**Status:** metadata-only stub. No full text — this is a paid, members-only commercial publication (unlike ISMAP and the Basic Act on Cybersecurity, which are government publications this repo could reasonably pursue further). Full ingestion would require a FISC membership or purchase.

## What's here

Public-domain-adjacent facts only: organization background, the ~300-control scale, and a third-party category grouping (Facility / Operational / Technical Standards, per an unofficial [Open Security Architecture mapping](https://www.opensecurityarchitecture.org/frameworks/fisc/) to NIST SP 800-53). That grouping is **not verified** against FISC's own clause numbering — treat it as a coarse orientation, not authoritative structure.

The paired SecID entry (`registry/control/jp/or/fisc.json`) mirrors this: three low-confidence category-level match_nodes, clearly marked as third-party-sourced.
